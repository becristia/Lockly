# Lockly 云同步手工测试方案与流程

**适用日期:** 2026-05-24  
**适用范围:** `Lockly` 前端 App、`backend-pass` 后端 API、后端管理台  
**测试目标:** 按步骤手工覆盖用户登录、设备管理、本地保险库、密钥同步、云下载、附件同步、冲突、紧急访问、管理台和安全边界。

---

## 1. 测试原则

- 只使用测试账号和测试密码，不使用真实密码、真实私钥、真实恢复包。
- 云账号密码和 App 主密码必须不同，用来验证“云账号不等于保险库解锁”。
- 后端只能看到密文和运维元数据，任何页面/API/日志都不能出现主密码、DEK、KEK、条目明文、完整密文。
- 每个用例记录结果：`PASS`、`FAIL`、`BLOCKED`，失败时记录截图、时间、账号、设备、后端日志片段。
- 如果构建或依赖下载连续失败 3 次，停止测试并记录网络问题。

## 2. 环境准备

### 2.1 测试设备

至少准备 1 台 Android 真机。完整云同步建议准备 2 个设备，优先级如下：

1. 设备 A: 当前真机 `23078RKD5C`。
2. 设备 B: 第二台 Android 真机或模拟器。
3. 只有一台设备时，用“清除本地数据后重新安装/重新初始化”模拟第二设备，但不能完全覆盖并发冲突体验。

### 2.2 启动后端

在 `backend-pass` 目录执行：

```powershell
$env:SECRET_KEY = [System.Guid]::NewGuid().ToString('N') + [System.Guid]::NewGuid().ToString('N')
$env:ADMIN_SESSION_SECRET = [System.Guid]::NewGuid().ToString('N') + [System.Guid]::NewGuid().ToString('N')
$env:DATABASE_URL = 'sqlite:///./backend_pass.db'
python -m uvicorn backend.app.main:app --host 127.0.0.1 --port 18000
```

验证：

```powershell
Invoke-RestMethod http://127.0.0.1:18000/health
```

期望：

```json
{"status":"ok"}
```

### 2.3 手机端口转发

```powershell
adb devices
adb -s 192.168.3.5:8888 reverse tcp:18000 tcp:18000
adb -s 192.168.3.5:8888 reverse --list
```

期望包含：

```text
tcp:18000 tcp:18000
```

从手机侧验证后端：

```powershell
adb -s 192.168.3.5:8888 shell "(echo -e 'GET /health HTTP/1.0\r\n\r\n'; sleep 1) | toybox nc 127.0.0.1 18000"
```

期望响应包含 `HTTP/1.1 200 OK` 和 `{"status":"ok"}`。

### 2.4 安装 App

在 `Lockly` 目录执行：

```powershell
flutter pub get
flutter build apk --debug --dart-define=LOCKLY_SYNC_BASE_URL=http://127.0.0.1:18000
adb -s 192.168.3.5:8888 install -r build\app\outputs\flutter-apk\app-debug.apk
adb -s 192.168.3.5:8888 shell monkey -p com.lockly.securebox 1
```

验证：

```powershell
adb -s 192.168.3.5:8888 shell pm list packages com.lockly.securebox
```

期望：

```text
package:com.lockly.securebox
```

### 2.5 创建管理台账号

在 `backend-pass` 目录执行：

```powershell
python scripts\create_admin.py
```

建议创建：

- 管理员: `admin-manual@example.test`
- 只读管理员: `readonly-manual@example.test`

密码使用测试密码，例如 `AdminPass123!Manual`，不要使用真实密码。

## 4. P0 主线测试

### P0-01 首次启动、语言和主密码创建

1. 可选清理本地数据：

   ```powershell
   adb -s 192.168.3.5:8888 shell pm clear com.lockly.securebox
   ```

2. 打开 App。
3. 检查默认语言是否为中文。
4. 进入设置或初始页面可达的语言入口，切换到 English。
5. 杀掉 App 后重新打开。
6. 确认语言保持 English。
7. 切回中文，再次重启 App。
8. 输入弱主密码，例如 `12345678`。
9. 输入两次不一致的主密码。
10. 输入强主密码 `MasterPass123!A` 并创建保险库。

期望：

- 语言切换持久化。
- 弱主密码被拒绝。
- 两次主密码不一致被拒绝。
- 创建保险库后进入已解锁的保险库界面。
- 云账号密码没有参与本地保险库创建。

### P0-02 解锁、锁定和隐私遮罩

1. 锁定 App 或重启 App。
2. 输入错误主密码。
3. 连续输入多次错误主密码。
4. 输入正确主密码。
5. 解锁后切到后台，再切回前台。
6. 等待自动锁定时间，或在设置中调短后验证。

期望：

- 错误主密码只显示泛化错误，不泄露细节。
- 多次失败出现退避或短暂阻止。
- 正确主密码能解锁。
- 后台时有隐私遮罩。
- 自动锁定后必须重新输入主密码。

### P0-03 本地条目增删改查

1. 在保险库首页新增条目，使用 3.2 的样例数据。
2. 保存后回到列表。
3. 搜索 `GitHub`、`manual-user`、`cloud`。
4. 进入详情页。
5. 检查密码默认隐藏。
6. 点击显示密码。
7. 复制用户名和密码。
8. 编辑条目，将标题改为 `GitHub Manual Edited`，密码改为 `ItemPass456!Manual`。
9. 保存并回详情页。
10. 查看密码历史。
11. 删除该条目。
12. 进入回收站。
13. 恢复该条目。
14. 再次删除，执行永久删除。

期望：

- 搜索能命中标题、网站、用户名、备注或标签。
- 密码默认不可见。
- 复制操作不改变条目内容。
- 编辑后列表和详情更新。
- 密码历史能看到旧密码记录或历史入口。
- 删除进入回收站，不直接物理删除。
- 恢复后条目回到列表。
- 永久删除后不可恢复。

### P0-04 密码生成器

1. 打开生成器页。
2. 调整长度、数字、符号等选项。
3. 点击生成。
4. 复制生成结果。
5. 使用“保存到条目”或从编辑页调用生成器，将生成密码保存到新条目。

期望：

- 生成结果符合所选策略。
- 复制不会导致明文长期驻留页面状态。
- 保存后新条目密码为生成值。

### P0-05 附件

1. 进入一个条目详情。
2. 添加 `manual-attachment.txt`。
3. 检查附件名称和大小显示。
4. 打开附件。
5. 删除附件。
6. 刷新或重新进入详情页。

期望：

- 附件元数据可见。
- 附件内容只能在本地解密后打开。
- 删除后附件不再显示。
- 后端管理台不显示附件明文或完整密文。

### P0-06 备份、导入和迁移

1. 设置页导出加密备份。
2. 保存或复制备份 JSON。
3. 尝试用错误主密码导入。
4. 用正确主密码导入。
5. 打开迁移向导。
6. 粘贴一段 CSV 样例，预览。
7. 确认预览页不暴露密码明文到不必要区域。
8. 执行导入。
9. 回保险库列表检查导入条目。

期望：

- 备份 JSON 是加密备份，不是明文条目。
- 错误主密码导入失败且不写入数据。
- 正确主密码导入成功。
- CSV 预览失败时会清理可见 CSV。
- 导入后条目进入本地加密保险库。

## 5. P0 云同步测试

### P0-07 云账号注册和登录

1. 设置页打开云同步区域。
2. 注册 `manual-a@example.test`，密码 `CloudPass123!A`。
3. 退出云同步。
4. 使用同一账号登录。
5. 打开后端管理台 `http://127.0.0.1:18000/admin/login`。
6. 登录管理员。
7. 进入用户列表和设备列表。

期望：

- 云账号注册成功。
- 登录成功后 App 显示已连接或类似状态。
- 管理台能看到用户和设备元数据。
- 管理台不显示 App 主密码。
- 云账号密码与主密码不同，不能用于解锁保险库。

### P0-08 首次上传同步

1. 在 App 中创建至少 2 个条目，其中 1 个带附件。
2. 设置页点击“同步到云”。
3. 在主密码弹窗中先输入错误主密码。
4. 再点击同步，输入正确主密码。
5. 观察同步状态。
6. 管理台打开同步事件、用户详情、设备列表。

期望：

- 同步前必须输入主密码。
- 错误主密码不会上传可解密数据。
- 正确主密码后同步成功。
- 管理台只看到密文长度、摘要、修订号、设备、时间等元数据。
- 管理台和日志不显示条目标题、用户名、密码、备注、附件明文。

### P0-09 云下载恢复

优先使用设备 B。没有设备 B 时，在设备 A 上先确认云端已有数据，再清理本地数据模拟新设备。

设备 B 流程：

1. 安装同一个 APK。
2. 执行 `adb reverse tcp:18000 tcp:18000`。
3. 首次启动并创建本地保险库，主密码使用 `MasterPass123!A`。
4. 登录云账号 `manual-a@example.test`。
5. 点击“从云下载”。
6. 输入错误主密码。
7. 再次下载，输入正确主密码。
8. 回到保险库列表检查条目和附件。

单设备模拟流程：

1. 确认云端已完成 P0-08。
2. 设置页执行清除本地保险库，或使用 `adb shell pm clear com.lockly.securebox`。
3. 重新打开 App。
4. 创建本地保险库，主密码仍使用 `MasterPass123!A`。
5. 登录云账号。
6. 执行“从云下载”。

期望：

- 错误主密码下载失败或无法导入。
- 正确主密码后恢复条目。
- 附件可恢复并打开。
- 后端没有主密码或主密钥。

### P0-10 双向同步

1. 设备 A 编辑 `GitHub Manual`，标题改为 `GitHub Manual A Edit`。
2. 设备 A 同步到云。
3. 设备 B 执行云下载或同步。
4. 检查设备 B 看到 A 的修改。
5. 设备 B 新增条目 `Device B Only`。
6. 设备 B 同步到云。
7. 设备 A 执行云下载或同步。
8. 检查设备 A 看到 B 的新增条目。

期望：

- 两端最终包含同一批条目。
- 修订号推进正常。
- 同步不会要求云账号密码解锁保险库，只要求本地主密码进行加解密。

### P0-11 设备管理

1. 设置页打开云设备列表。
2. 检查设备名称、平台、客户端版本、最后同步时间。
3. 重命名当前设备。
4. 管理台设备页确认名称变化。
5. 在设备 B 登录同一云账号并同步。
6. 设备 A 的云设备列表应出现设备 B。
7. 在设备 A 或管理台撤销设备 B。
8. 在设备 B 再执行同步。

期望：

- 设备列表显示元数据，不显示本地密钥。
- 重命名成功并同步到管理台。
- 被撤销设备不能继续同步。
- 设备撤销产生审计日志。

## 6. P1 冲突、删除和异常测试

### P1-01 同一条目并发冲突

需要设备 A 和设备 B。

1. 两台设备都完成一次同步，确保同一条目修订一致。
2. 设备 A 断网或暂时不让设备 B 同步。
3. 设备 A 编辑 `GitHub Manual` 的密码为 `ConflictPassA123!`。
4. 设备 A 同步到云。
5. 设备 B 在未先下载 A 更新的情况下，编辑同一条目的密码为 `ConflictPassB123!`。
6. 设备 B 同步到云。
7. 打开设备 B 同步状态、设备 A 同步状态。
8. 管理台进入冲突页。

期望：

- 后端返回冲突，不自动覆盖。
- 设备 B 本地修改不能被静默丢弃。
- 管理台冲突页只显示 item id、修订号、时间、用户、设备等元数据。
- 管理台不显示 `ConflictPassA123!` 或 `ConflictPassB123!`。

### P1-02 远端软删除同步

1. 设备 A 删除一个条目。
2. 设备 A 同步。
3. 设备 B 下载或同步。
4. 设备 B 检查该条目是否不再出现在活动列表。
5. 设备 B 检查回收站或删除状态。
6. 管理台同步事件检查删除事件。

期望：

- 删除以 tombstone/软删除形式同步。
- 其他设备不会继续显示活动条目。
- 管理台能看到删除元数据，不显示明文。

### P1-03 网络中断

1. 删除 adb reverse：

   ```powershell
   adb -s 192.168.3.5:8888 reverse --remove tcp:18000
   ```

2. 在 App 中执行登录、同步、下载、设备列表。
3. 恢复 reverse：

   ```powershell
   adb -s 192.168.3.5:8888 reverse tcp:18000 tcp:18000
   ```

4. 重试登录和同步。

期望：

- 网络断开时显示可理解的失败状态。
- App 不崩溃。
- 本地保险库仍可正常浏览、编辑、锁定、解锁。
- 网络恢复后可以重试成功。

### P1-04 Token、登出和会话撤销

1. 云账号登录后执行同步。
2. App 中退出云同步。
3. 尝试同步。
4. 重新登录云账号。
5. 管理台用户详情中撤销该用户会话。
6. App 再次尝试同步。

期望：

- 登出后不能继续同步。
- 重新登录后恢复同步能力。
- 管理台撤销会话后，旧 refresh token 不能继续换 token。
- App 需要重新登录云账号。

### P1-05 本地安全和剪贴板

1. 复制密码。
2. 立即粘贴到安全测试输入框，确认复制内容正确。
3. 等待剪贴板清理时间。
4. 再次粘贴。
5. 复制用户名，等待密码清理时间。

期望：

- 密码复制后会按设置清理。
- 用户名复制不应被密码清理逻辑误删。
- 后台切换不会在最近任务预览中显示敏感内容。

## 7. P1 紧急访问测试

### P1-06 联系人和授权包

建议使用用户 A 作为所有者，用户 B 作为接收者。

1. 用户 B 登录并进入紧急访问页面。
2. 生成接收者公钥和指纹。
3. 确认私钥只显示在本地，不上传到管理台。
4. 用户 A 登录，进入紧急访问。
5. 添加联系人，填写用户 B 邮箱、公钥、指纹、标签。
6. 创建授权包，设置等待时间。
7. 用户 B 刷新紧急访问列表。
8. 用户 B 接受授权。
9. 用户 B 请求访问。
10. 等待时间满足后，用户 B 下载授权包。
11. 用户 A 撤销授权或联系人。

期望：

- 联系人和授权流程可完成。
- 授权包、请求消息、AAD 都以密文或安全元数据形式出现。
- 私钥不进入后端。
- 撤销后不能继续下载。
- 管理台只看到安全元数据和审计事件。

## 8. P2 后端管理台测试

### P2-01 登录、语言和导航

1. 打开 `http://127.0.0.1:18000/admin/login`。
2. 检查默认中文。
3. 切换 `?lang=en`。
4. 登录管理员。
5. 访问 Dashboard、Users、Devices、Sync Events、Conflicts、Security、Audit Logs。
6. 登出。

期望：

- 未登录访问后台页面会跳转登录。
- 中英文切换不翻译用户邮箱、设备名等业务数据。
- 登出后不能继续访问后台页面。

### P2-02 用户管理

1. 用户列表搜索 `manual-a`。
2. 打开用户详情。
3. 查看设备、会话数量、最近登录时间。
4. 禁用用户。
5. 使用 App 尝试云同步。
6. 恢复用户。
7. 撤销用户所有会话。

期望：

- 禁用用户后无法继续使用 access/refresh token。
- 恢复用户后可重新登录。
- 撤销会话产生审计日志。
- 用户详情不显示密码 hash、主密码、DEK、KEK、条目明文。

### P2-03 设备页

1. 设备页按 active/revoked 筛选。
2. 搜索设备名。
3. 撤销一个非当前测试关键设备。
4. 检查分页。

期望：

- 筛选和搜索可用。
- 撤销设备有二次确认。
- 撤销后设备状态为 revoked。
- 只读管理员不能撤销设备。

### P2-04 同步事件和冲突页

1. 先执行一次上传、下载、删除、冲突测试。
2. 打开同步事件页。
3. 使用关键词、事件类型、时间范围筛选。
4. 打开冲突页。
5. 使用关键词、状态、时间范围筛选。

期望：

- 能看到 push、pull、delete、conflict 等事件。
- 筛选结果准确。
- 页面不显示完整 ciphertext，不显示条目明文。
- 冲突未自动解决时保持可追踪状态。

### P2-05 安全说明页

1. 打开 Security 页面。
2. 检查零知识边界说明。
3. 检查页面是否明确说明后端不能解密本地保险库。

期望：

- 页面说明主密码不同步。
- 页面说明后端只存密文和元数据。
- 页面不含误导性“云端可恢复主密码”之类文案。

## 10. 安全边界专项检查

以下检查在每次完整手工测试后执行。

### S-01 后端数据库检查

在 `backend-pass` 目录，使用 SQLite 工具或 DB Browser 查看 `backend_pass.db`。

检查：

- `users.password_hash` 不是明文云账号密码。
- `admin_users.password_hash` 不是明文管理员密码。
- `vaults` 不包含 App 主密码明文。
- `vault_items` 不包含条目标题、用户名、密码、备注明文。
- `vault_blobs` 不包含附件明文。
- `auth_sessions.refresh_token_hash` 不是 refresh token 明文。

### S-02 管理台页面源码搜索

在浏览器中分别打开管理台页面，使用页面查找搜索以下测试明文：

- `MasterPass123`
- `ItemPass`
- `manual note should stay local`
- `manual attachment test content`
- `DEK`
- `KEK`

期望：

- 不出现主密码、条目密码、备注、附件内容。
- `DEK`/`KEK` 只应出现在安全说明文字里，不能作为真实密钥值出现。

### S-03 后端日志检查

查看 uvicorn 输出和错误日志。

期望：

- 请求日志不包含 Authorization token。
- 日志不包含主密码。
- 日志不包含条目明文或完整密文。
