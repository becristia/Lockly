## 未通过项目

### P0-08 首次上传同步

**状态:** FAIL  
**执行人:** 用户手工测试  
**复核时间:** 2026-05-24 23:04:00 +08:00

确认范围：

- App 执行云同步后，后端管理台有同步时间，但复核确认该时间来自 vault 初始化或 pull/blob_pull 事件，不能代表条目上传成功。
- 后端日志显示 `POST /sync/push-vault` 返回 `422 Unprocessable Entity`。
- 后端数据库显示 `vaults=1`，但 `vault_items=0`、`vault_blobs=0`。
- 前端输入主密码后直接跳到解锁/密码库页面，未停留在设置页展示明确同步结果。

遗留问题：

- `SYNC-001`: 首次上传带条目/附件的 vault 时，`/sync/push-vault` 被后端 422 拒绝，导致云端只有 vault 元数据，没有条目和附件。
- `UX-001`: 云同步主密码确认完成后应留在设置页或云同步区域，并展示同步成功、失败或冲突状态；当前跳转到解锁/密码库页面，容易让用户误判同步是否完成。

结论：

- P0-08 未通过，需要后续修复 `SYNC-001` 和 `UX-001` 后回归。
- 按当前测试安排，暂跳过 P0-09 到 P1，转入 P2 后端管理台测试。

结论：

- P0-09 暂时被 P0-08 上传失败阻塞，不按主密码错误处理。
- 修复并回归 P0-08 上传成功后，再重跑 P0-09。
